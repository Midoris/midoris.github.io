Qixxit - Train Bus Flight

iPhone and iPad app

The app was built from scratch by me and my teammate while I was working in QT Mobilit√§tsservice GmbH (Qixxit) in Berlin.

  • Apple Pay
  • Completely written in Swift
  • VIPER architecture
  • SocketServer
  • RESTful API
  • Functional Programming with ReactiveSwift and RectiveCocoa
  • Google maps with custom drawings
  • Push Notifications
  • Firebase for storage and remote config
  • Fabric and Circle CI for internal release

Our iOS app was featured multiple times by Apple and we were partners with Apple in launching Apple Pay in Germany. We also got invited to the closed Apple Developer Event in Berlin in 2018.

The most challenging and interesting features I developed was the implementation of sorting and filtering of itineraries.

The challenge was that we were using socket connection with backend for receiving results, meaning results were coming constantly throughout some time. Filtering and sorting must be done on the fly any time we got a new bunch of results and also any time user changes his filter and sorting options, which can happen while we are still receiving results.

I used functional programming with ReactiveSwift to solve this problem. I made all filters and sorters to be properties that emit en event on change and subscribed them all to one listener that reacts with recalculation and UI update.

Sorting and filtering was successfully implemented. With time the number of filters and sorters were increased, code was easily maintainable and understandable for other team members.

Cortex: notes organizer

iPad app

My own app, live in the App Store.

The goal of Cortex app is to structure and organization your notes to make them easy to find.

It uses a mixed organizational structure: Folders and Tags. So the note can be in a few folders simultaneously, which makes it very easy to find, and no structuring by hand needed.

  • Completely written in Swift
  • VIPER architecture
  • Text analysis with IBM Watson NaturalLanguageUnderstandingV1 API
  • Advanced search with NSLinguisticTagger
  • Core Data
  • Consumable In-App Purchases

For tagging the notes Natural Language Understanding API from IBM Watson was used. I am using Concept, Entity, and Category to get the best tags options to the user.

For an advanced search, NSLinguisticTagger is used with Lemma and Language schemes. Which means it will find the same word in a different forms, like say you have entered a word `walk` and it will find `walking` and `walked`. Also, it will find the same word in different languages.
